# 播客转录文本结构化指南

## 基础结构要求

1. 文章框架
- 标题：[原标题 + 副标题]
- 摘要：200字内，突出3-5个核心观点
- 思维导图：生成 Mermaid 代码
- 目录：自动生成章节导航
- 正文：分5-10个主要章节
- 结语：总结与延伸思考

2. 内容处理规范
- 将对话转为叙述文体
- 保留关键引用与案例
- 添加必要的上下文说明

## 重点呈现元素

3. 核心内容可视化
- 💡 关键观点信息框
- 📊 数据可视化(如适用)
- ⚡️ 重要引言区块
- 🔍 深度探讨模块
- 📘 专业术语表

4. 特殊处理标记
- 「原文引用」：保留精彩原话
- [编者注]：补充说明

## 质量控制清单

5. 内容检查项
- [ ] 逻辑连贯性
- [ ] 术语准确性
- [ ] 数据可信度、记住不要伪造播客中不存在的信息或事实
- [ ] 观点完整性
- [ ] 表达专业性

6. 格式检查项
- [ ] 标题层级
- [ ] 段落划分
- [ ] 图文配比
- [ ] 引用规范
- [ ] 术语注释

## 输出要求
使用简体中文输出一篇高质量的专业技术类微信公众号文章，不要局限文本过长。
